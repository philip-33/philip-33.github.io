Full Stack vs. Front-End


Getting Started


Types of Web Developers
* Front-end developers deal with the parts that users interface with and use HTML/CSS, javascript
* Back-end developers focus on parts of the website that run on servers and handle data processing in the background
* Web developers are expected to have an understanding of the “full stack” which refers to both front and back-end together, but generally specialize in one or the other.
The role of HTML
* Important for everyone to know, defines structure and data on a webpage
* Front-end developers are required to understand the Information Architecture (IA)
* Tags that create the structure of a page and the IA work together to create the DOM
* Foundational to all roles in this field
The role of CSS
* Drives the experience of the website and how content is displayed to the user
* Defines how the DOM is translated into visual form
* CSS Zen Garden is an example of a single HTML document displayed in different ways
The role of Javascript
* Javascript interacts with the DOM: Handling events, load/unload media, create/modify/delete page elements
* Used as part of the interface with front-end, also handles back-end tasks like compression or workflow management
Back end essentials
* Each company has a different stack (set of programs/libraries/tools) that they use
* All web developers need to know at least one stack for working with websites (video recommends: PHP + SQL)
Servers
* This hosts the files for the site and manages presentation to the user.
* Back-end developers manage the site and services, front-end developers manage the interface/experience
* Install your own stack to practice at home
Server-side scripting
* Main languages are PHP, nodeJS, .NET, ASP.NET, Ruby, Python
* Important to know how to use more than 1 platform. Some are agnostic and run on multiple systems. Others are specific, like ASP.NET runs only on Windows.
Database languages
* SQL creates tables, like spreadsheets, that link data on tables with relationships
* NoSQL represents a new direction for databases, as they lack a traditional relational structure

Learning and Training


Train for the industry
* Multiple ways to learn to be a web developer, 70% of whom are self-taught
* Read documentation, examine other developer’s code, explore new ideas
* Making mistakes makes you a better developer
Traditional education
* College education means that developers are able to think more deeply about subjects
* General education provides a wide base of experience and higher math classes can help with some aspects of development
* These skills provide a higher overall earning potential vs bootcamps and self-study
Coding bootcamps
* Quickly teach common skills for development, and 88% placement rate within a few months
* 10-12 weeks of intense training.
* 79% of camp students already have a bachelor’s degree
Other learning options
* Online learning is a key way to keep up with a rapidly changing industry
* Some sites are highly focused such as a Slack run by a company to provide a community with access to their developers, or broad like stackoverflow where people can ask questions and different answers are voted on by the other forum members.
Social learning opportunities
* ~30% of all developers got their jobs from a friend, <20% got them by applying through a website.
* Conferences and meetups are ways to get involved socially, but be genuine and find events that interest you
Core developer tools
* 3 tools to master: Your editor (IDE), Terminal (CLI), Browser
* Sites can be built with these alone, and they must be mastered
Other essential tools
* Data visualization is helpful for dealing with databases (phpMyAdmin)
* Version control is done with git/github
* Workflow tools help with common processes for making websites (minifying, transpiling)
* Frameworks help you build websites with sets of prebuilt code
Core concepts
* Employers are more interested in developers that are able to make sites that are more accessible to larger numbers of people
* Employers appreciate developers that are familiar with Software Engineering workflows such as agile development processes that work in 2-week springs
* Test-driven development creates code and tests the code at the same time
Business and leadership skills
* Communication skills (non-confrontational) are critical for necessary team work
* It includes presenting and defending your ideas and work is also necessary
* Github with code examples is critical
* Salary negotiation skills are mandatory

Getting a Job


Types of jobs
* True full-stack dev is a unicorn
* Remote devs have higher job satisfaction
* More talented developers will have an easier time finding well paying, interesting work
Maximize compensation
* $65K is average web developer salary
* $100K is median developer salary with 5 years experience (full-stack), $97K for front end
* For various levels of education, a small amount of experience leads to $100K+ salary
Land a job
* Referrals, no really.
* Recruiters via meetups
* Find people in a company that you want to get hired with
