Git branch command

This was pretty straightforward. I even made a mistake editing a file so I was able to try reverting commits, and deleting the branch. Even after a single go through the process it's clear that the command line is easier to use than the website or desktop client because it auto-manages the file versions for you. When checking out different branches I was surprised to see the file content in VSCode revert to the version before my most recent changes, but then I understood that's how it was intended to work.

Looking for some more information, I found a page from the GitHub documentation that explains how to pause work on one branch to make changes to another branch, then resume working on the previous branch while keeping commits separate. It doesn't appear to use much beyond what was covered in the tutorial, just a few extra steps for dealing with the new issue branch.
