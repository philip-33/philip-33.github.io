Getting Started with GitHub

Just this summer I dove more deeply into GitHub and learned how to use it. I normally use the github desktop client to manage my repos, but I expect to move to the command line before the end of this class. One of the things that I was interested in was the professional aspect of GitHub commits. That is, when working on larger projects, how much is in a single commit (it should be atomic in the sense that it is a single feature or change, even if it spans a large amount of code), what is a good name for the commit (keep it simple to keep the size of the message down, but be descriptive so itâ€™s apparent what the commit covers without looking at the code). Also, the description should have details about the code and its major features. After using these guidelines for the commits, my commit history is more meaningful and it's easier to see my development process.
