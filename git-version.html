Semantic Versioning

This is a really clever system that has come up around the need for a clear way to understand versioning. Not only does it quickly convey the relationships between versions in a meaningful way, it also gets developers thinking more critically about their applications and how they write and release software.

The SemVer website has some interesting questions in the FAQ that directly address the software development process. The question regarding rapid major version increases because of minor API changes is countered by suggesting against frivolous API revisions because the cost of API upgrades could be significant to users.

For a program that uses semantic versioning, I found Burp Suite. It's a large set of tools designed to help with network security testing:

Burp Suite by Portswigger

As of August 31, 2017, Burp Suite is version 1.7.27. Their website shows December 2008 with their initial full public release of version 1.2, and January 2009 had their first patch with 1.2.01.
