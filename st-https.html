HTTPS

This is a really good video. It’s target audience is website admins and it has a lot of info that is new to me. One thing that was especially interesting was the part where HTTPS/2 stores session ID’s to minimize the amount of authentication necessary between a client and server. Session hijacking was covered in the security classes I’ve taken, but it was in a higher-level context and I didn’t understand the context around it and how it fit into regular web use. Now it’s clear the ID is used to help drastically speed up web transactions, and the associated trade-off between security and functionality is apparent.

Instead of stealing your username and password, an attacker can get your session IDs for a given site and use them in another browser. On seeing the session ID, the website would give the attacker full access to your account. This also informs why some websites like amazon will ask for your password again before major account changes such as changing payment methods or passwords.
