Web Technology Fundamentals 1


Introduction to the Web


Understanding how the web works
* Internet and web are not the same thing
* There are many types of web servers that specialize in different types of data
Working with clients and servers
* Clients request data from servers with a URL that is translated by DNS into an IP
* Static sites process most of the data on the client, while dynamic sites process some data on the server and present it to the client
The DNS and why it matters
* DNS is a global, distributed database with authoritative root servers
* Each TLD (.com, .edu, etc) has its own DNS server
Internet protocols
* The Internet Protocol Suite (TCP/IP) is a collection of protocols that includes HTTP, FTP, and others to allow sharing of information
* The suite can be broken down into multiple layers that separate protocols based on whether they present information to a user, manage addressing, facilitate transport between network nodes, or connect individual devices
Exploring HTTP
* FTP: used for files, SMTP: used for mail, RTP: used for video
* Stateless, so other methods need to be used to save data between transmissions (cookies, javascript, etc)
Anatomy of a URL
* As long as it is recognized by the server, any protocol can be used
* Scripting can be used to request data from the server using the URL
How browsers work
* Servers can serve any type of data, but it can only be viewed in the browser if the browser supports that file type
* Different browsers, and even different versions of the same browser, can have different rendering engines
Web standards and the W3C
* W3C started in 1994 and issued recommendations for web technologies
* The Web Standards Project branded these recommendations as standards :/
Web server basics
* Web servers usually use a shared hosting environment where multiple sites are hosted on the same server, but dedicated options are available.
* LAMP: Linux, Apache, MySQL, PHP (or Perl, or Python)

Front-End Technology


Front-end design
* This deals with the presentation of a website to the user
* Front-end development is a slightly different focus than Front-end design, but they have multiple skills in common and are sometimes used interchangeably
HTML, language of the web
* HTML 2.0 was released in 1995 and was the first release intended to be a ‘standard’
* XML 2 was so broken that a group separate from the W3C formed and drafted HTML5
Structuring HTML
* Organized with tags that identify the different parts of a web page’s structure
* CSS can be added to each tag individually, applying styles to groups of tags at the same time depending on their function and the needs of the site
Controlling presentation through CSS
* Cascading is a major feature of CSS that allows for detailed manipulation of content in HTML tags based on the tags themselves, classes, and ID’s
* Having style (CSS) and structure (HTML) as separate components allows for greater portability and less dependency
Client-side scripting with JavaScript
* Frameworks are libraries that already have functions and objects built-in
* Start with native JavaScript, then branch out to frameworks
