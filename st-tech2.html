Web Technology Fundamentals 2


Front-End Technology (continued)


Common Image Types
* Need to find a balance between image quality and file size
* Knowing the specific capabilities and trade-offs of each format allows devs to make the best decision for the site
What’s an API
* At its core, an API is a way for a service to be used on other sites or in other programs by providing a set of instructions/services for developers to use
* APIs are not the same as the sites themselves
HTML5 APIs
* After being first developed by the WHATWG, W3C broke the HTML5 API into multiple parts that allowed them to develop separately
* HTML5 APIs include Media, Drag-and-Drop, and App Cache. Separate APIs include Canvas and Geolocation
Web Fonts
* Temporary fonts used by CSS @font-face rule
* After legal and security issues were resolved, font creators have assisted in the adoption of web fonts

Back-End Technology


Server-side scripting
* Anything that happens on the server: registering new users, confirming purchase, etc
* SS Scripts run on the server before the page is presented to the user. Load is on the server, but scripts are hidden from the end user
The importance of PHP
* PHP became a key feature of websites when combined with Apache and MySQL
* PHP is a scripting language, and when a webserver sees a file with the .php extension, it understands that processing is required before the page can be presented to the user
Other popular server languages
* Includes: Java Server Pages, VB.NET, Coldfusion, Python, Ruby
* Choosing a language depends on cost, developer availability, and ease of use
Dealing with data
* On the browser, it’s cookies or key-value pairs. With JavaScript, XML and JSON files are used. For larger applications, a database is necessary.
* Normally only server side, Indexed DB and Web SQL are browser side databases
* Two types of databases: Relational and Non-relational. Relational DBs are organized by a schema that describes how tables are linked and related to each other. NoSQL DBs are flat and easier to index.
Using SQL databases
* Simple syntax: SELECT, INSERT, UPDATE, DELETE
* Multple versions of SQL, often proprietary
Content management systems
* Designers can create templates that media and other content can be applied to the structure.
* Often uses WYSIWYG style editing so that non-technical end users can manage the site after the development is complete.
* Popular CMS: Joomla, Drupal, Movable Type, ModX, Radiant, WordPress, and Expression Engine
Content delivery networks
* CDN is way to deliver content via a distributed network of servers worldwide
* Specific parts of a site can be offloaded to a CDN to improve load times overall as well as reduce your own servers overhead.
Cloud services
* Using someone else’s computer as a way to have files/content available wherever there is an internet connection
* Built with the same tech used for websites (Ruby, .NET, Java, JavaScript) but are architected differently and need to be scalable
What is GitHub
* Online distribution service providing version control that was built around Git
* The forking and merging process that is part of GitHub has helped the FOSS community grow quickly

Assembly Technologies


JavaScript libraries
* Pre-written javascript code that makes it easier to do common tasks
* Increases capabilities of the site and the developer, but also increases overhead to load the site initially. Important to know when a library is appropriate
Frameworks and boilerplates
* Collection of prebuilt HTML/CSS/JS files that allow quick site development
* Comprehensive frameworks are a whole-page style that include everything a developer will likely ever need. Other minimalist frameworks only provide the basics and leave most of the styling up to the developer.
* Not always right for every project
CSS preprocessors
* Extends CSS with new syntax, then compiles the script to regular CSS
* Can add functionality to CSS and handle browser inconsistencies. Can also create problems when working with other developers and adds a layer of complexity.
