Visual Studio Code for Web Developers


Chapter 1 - Overview of VS Code


VSCode UI
* Left sidebar has basic tools: Explorer, search, git (if enabled), debug, extensions
* Command Palette (CTRL + Shift + P) provides access to all VSCode commands
Basic editing
* If there is no selection, cut and copy work on the whole line
* VSCode autocompletes where it can, including quotes, brackets, braces, and parentheses
The integrated terminal
* The integrated terminal runs with the same permissions as VSCode
* Multiple terminals can be opened using the dropdown
Integrated Source Control
* With Git connected, VSCode shows when files are different from the repo
* VSCode can show current and staged files side-by-side, and can commit staged changes with a commit message

Chapter 2 - Customizing VS Code


Configuring preferences
* By selecting preferences, we’re show a default file that is read-only and a custom file that overwrites user or workspace defaults with custom values. The defaults are searchable.
* Most functions of the program are controlled in the “Editor” and “Workbench” sections
Setting keyboard shortcuts
* All shortcuts are editable, but some only happen within certain contexts (window focus, for instance)
* Edited keyboard shortcuts are saved in a .json file, like other preference changes
Using extensions
* Extensions can be found via browser or within VSCode itself. The search feature is powerful.
* Extensions can add arrays of features for specific languages, and VSCode can recommend extensions based on the files you are working with
Essential extensions
* This list is focused on web apps, but there are many extensions for other languages
* Extensions shown: HTML5 Snippets, JQuery Snippets, Beautify, Express (web server), REST client extension
Working with snippets
* Snippets are provided by VSCode or extensions, and are usually completed by starting the command/tag and a popup will suggest completion options.
* Snippets can be configured and customized, again with json files.
Color and Icon themes
* Themes cover the colors of the windows, backgrounds, and text syntax highlighting
* Icon themes change the icons used for files in the explorer. A good icon theme will make quickly identifying files easier

Chapter 3 - Using the Editor


Files, folders, and projects
* VSCode treats individual folders as projects, and can parse existing project structures
* When a filename is in italics in the tab bar, it is in preview mode, which means the tab will be reused to show other selected files. After saving, the filename will become normal and the tab will remain until closed.
* Right-clicking on a file will allow opening a terminal directly to that folder
Side by side editing
* Right-click on a file and select “open on the side” to see the same or different files side by side
* The split can be horizontal or vertical, and each viewport can have its own set of tabs
IntelliSense
* AKA code completion, and built into VSCode. It assists with programming by suggesting common completions for characters that have been typed.
* In addition to completion, it can also show function parameter lists to ease the burden of remembering detailed and/or complicated function parameters
Finding and replacing
* CTRL+F begins the search function. The find panel also allows search and replace, and the edit menu allows searching across multiple files
* CTRL+SHIFT+J allows for more detailed searching with wildcards

Chapter 4 - Advanced Editing


Editor features
* Hot exit saves unsaved files, enable autosave.
* VSCode can update the formatting of a file on save based on the formatting rules provided by VSCode or extensions. This is disabled by default.
Web code editing
* VSCode can quickly comment out selected code with CTRL+/ and uses the correct method based on the file type. Additionally, it can assist with the generation of documentation
* Markdown features allow for quick prototyping and works in multiple file types. Also, Emmet is a common tool for autocompleting HTML snippets using a shorthand.
Navigating code
* CTRL+G jumps to a specific line, CTRL+Shift+o jumps to a symbol (id, class name, etc.)
* Using CTRL+P (quick open), VSCode shows a list of recent files to choose from
Using multiple selections
* VSCode has CTRL+D for specific selection of the same portions of code on different lines. Successive CTRL+D’s will select addition instances of the same code. This allows for editing multiple lines of code at one time.
* CTRL+ALT+up arrow
